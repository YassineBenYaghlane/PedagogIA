The three categories the FWB uses to classify every content item in a référentiel. Defined in the Code de l'enseignement (art. 1.3.1-1) and quoted in [[sources/referentiel-mathematiques]].

## Definitions

- **Savoir** — a fact, set of facts, definition, concept, theory, model, or linguistic tool. *What one knows.*
- **Savoir-faire** — a procedure, gesture, technique, or resolution schema, standardized and automated by practice. *What one can do.*
- **Compétence** — the aptitude to mobilize a set of savoirs, savoir-faire, and savoir-être to accomplish tasks — especially novel ones requiring analysis. *What one can solve.*

The distinction between savoir-faire and compétence hinges on **novelty / transfer**: applying a learned procedure to a familiar case is savoir-faire; choosing the right one for an unfamiliar situation is compétence.

## Attitudes (savoir-être)

Attitudes are treated implicitly: either embedded inside a specific savoir-faire (e.g. rigour in experimental protocol) or lifted into the transversal *domaines 6, 7, 8*. No separate "savoir-être" column on primary math pages.

## Why this matters for PedagogIA

This is a **ready-made three-tier skill model** we can mirror in our DAG:

1. **Knowledge nodes** (savoirs) — "knows the symbols +, −, ×, ÷"
2. **Procedural nodes** (savoir-faire) — "performs the écrit multiplication algorithm with a 1-digit multiplier"
3. **Problem-solving nodes** (compétences) — "solves a word problem involving the four operations and verifies plausibility"

Each level unlocks the next. The référentiel tags every attendu with its category, so we can lift the classification directly when seeding the skill tree.
